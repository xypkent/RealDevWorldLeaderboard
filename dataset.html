RealDevBench Dataset

RealDevBench is a benchmark dataset designed to evaluate AI agents' capabilities in real-world application development scenarios. It includes various application development tasks, from frontend design implementation to backend API development, covering multiple aspects of software development.

Dataset Overview

The RealDevBench dataset consists of real-world application development tasks derived from actual development projects and challenges. Each task provides detailed requirement descriptions, expected output results, and evaluation criteria.

Task Diversity

Covers frontend, backend, full-stack, mobile applications, and other development task types, reflecting the complexity of real development scenarios.

Multi-level Difficulty

Tasks range from simple to complex, enabling comprehensive evaluation of AI agents across different challenge levels.

Detailed Evaluation Criteria

Each task comes with clear evaluation criteria covering everything from functionality implementation to code quality and performance.

Complete Development Process

Task design encompasses the complete development process from requirement understanding to design, implementation, and testing.

Dataset Features

  • Authenticity: Tasks come from real projects, not artificially constructed simple examples
  • Comprehensiveness: Covers multiple aspects of software development and technology stacks
  • Standardization: Unified description format and evaluation standards
  • Extensibility: Easy to add new tasks and evaluation dimensions
  • Practicality: Focuses on practical application scenarios rather than purely academic tests

Data Structure

The RealDevBench dataset is organized in a structured JSON format for easy processing and extension. Each task includes the following main fields:

{
  "task_id": "FE-001",
  "title": "Responsive Product Display Page",
  "category": "Frontend Development",
  "difficulty": "Medium",
  "description": {
    "background": "...",
    "requirements": ["...", "..."],
    "constraints": ["...", "..."]
  },
  "resources": {
    "api_docs": "...",
    "design_files": ["...", "..."],
    "starter_code": "..."
  },
  "evaluation": {
    "functional_criteria": ["...", "..."],
    "non_functional_criteria": ["...", "..."],
    "scoring_weights": {
      "design_understanding": 0.3,
      "code_implementation": 0.5,
      "debugging": 0.2
    }
  },
  "expected_output": {
    "description": "...",
    "examples": ["...", "..."]
  },
  "metadata": {
    "created_at": "2024-05-15",
    "version": "1.0",
    "tags": ["React", "Responsive", "E-commerce"]
  }
}

Main Data Fields Explanation

Field Description
task_id The unique identifier for the task, formatted as "Category-Number"
title A short descriptive title for the task
category The development category the task belongs to (frontend, backend, full-stack, etc.)
difficulty The difficulty level of the task (easy, medium, hard)
description Detailed description containing task background, requirements, and constraints
resources Task-related resources, such as API documentation, design files, etc.
evaluation Task evaluation criteria and scoring weights
expected_output Description of expected task completion results and examples
metadata Task metadata including creation time, version, and tags

Example Tasks

The following are some example tasks from the RealDevBench dataset, showcasing different types and difficulty levels of development challenges:

Task: Responsive E-commerce Product List Page

Frontend Development Medium Difficulty React
Background

An e-commerce website needs a product listing page that can display product information and provide filtering, sorting, and pagination functionality. The page needs to adapt to different screen sizes, from mobile devices to desktop.

Main Requirements
  • Display product cards, including image, name, price, rating, and short description
  • Implement filtering by category, price range, rating, etc.
  • Provide sorting options (price low to high, price high to low, rating)
  • Implement pagination or infinite scroll loading
  • Responsive design, adapting to mobile and desktop
Evaluation Dimensions
  • Design Understanding: Grasp and implementation of responsive design principles
  • Code Implementation: Quality of component structure, state management, and data processing
  • User Experience: Interaction design, loading performance, and usability

Task: RESTful API Design and Implementation

Backend Development Hard Node.js
Background

Design and implement a RESTful API for an online learning platform supporting course management, user registration, and learning progress tracking. The API needs to support authentication, authorization, data validation, and error handling.

Main Requirements
  • Design an API architecture following RESTful principles
  • Implement user authentication and role-based authorization
  • Design data models and relationships
  • Implement complete CRUD operations
  • Add appropriate error handling and logging
  • Implement basic performance optimizations (e.g., caching)
Evaluation Dimensions
  • API Design: Interface consistency, resource naming, status code usage
  • Security: Authentication and authorization implementation, input validation, security best practices
  • Code Quality: Structural organization, error handling, test coverage

Usage Guide

Follow these steps to use the RealDevBench dataset:

  1. Download the dataset from GitHub or Hugging Face
  2. Set up your evaluation environment according to the documentation
  3. Choose a subset of tasks to evaluate or use the complete dataset
  4. Run the evaluation scripts, connecting your AI agent to the evaluation framework
  5. Analyze the results and generate performance reports

Detailed usage guides and API documentation can be found in the GitHub repository.
